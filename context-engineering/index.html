Context Engineering이란?

"프롬프트 엔지니어링"의 다음 단계. 단순히 질문을 잘 하는 것을 넘어,
AI가 작업할 때 참조할 전체 맥락을 체계적으로 설계하는 것.

📝
Prompt Engineering
사용자가 AI에게 보내는 개별 질문을 잘 작성하는 기술. "이렇게 물어봐라"에 초점을 맞춘다. 일상적인 LLM 사용에서 자주 언급된다.
🧠
Context Engineering
AI가 작업할 때 참조하는 전체 맥락을 설계하는 기술. 설정 파일, 코드 컨텍스트, 도구, 히스토리, 예제까지 포괄한다. 산업 수준의 AI 애플리케이션에 필수적이다.
📄
정적 컨텍스트
CLAUDE.md, AGENTS.md, .cursorrules 같은 설정 파일. 프로젝트에 한 번 작성해두면 매 대화에서 자동으로 로드된다. 팀 전체가 공유할 수 있다.
⚡
동적 컨텍스트
RAG, 도구 호출 결과, 대화 히스토리 같은 실시간 정보. 컨텍스트 윈도우의 나머지 공간을 상황에 맞게 채운다. 요약과 압축이 핵심이다.

Context Window를 채우는 7가지 요소

Karpathy가 정의한 컨텍스트 윈도우의 핵심 구성요소

1
Task Description
시스템 프롬프트, 사용자 메시지 등 작업 지시. CLAUDE.md나 AGENTS.md가 여기에 해당한다.
2
Few-shot Examples
원하는 출력 형식을 보여주는 예제. 코딩 컨벤션, 커밋 메시지 스타일 등을 예시로 전달한다.
3
RAG (Retrieval-Augmented Generation)
관련 문서, 코드, API 레퍼런스 등을 검색해서 주입. 벡터 DB나 코드 인덱싱이 여기에 해당한다.
4
Multimodal Data
스크린샷, 다이어그램, 로그 파일 등 텍스트 외의 데이터. 디자인 시안이나 에러 스크린샷을 전달한다.
5
Tools
AI가 사용 가능한 도구 정의. 파일 읽기/쓰기, 터미널 실행, 웹 검색 등의 함수 스키마를 포함한다.
6
State & History
대화 히스토리, 이전 도구 호출 결과, 현재 작업 상태. 긴 대화에서는 요약(compaction)이 필수적이다.
7
Compacting / Summarization
컨텍스트 윈도우가 가득 차면 오래된 정보를 요약해서 압축. 중요한 정보는 유지하면서 공간을 확보한다.
