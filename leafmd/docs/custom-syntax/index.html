Leaf에서만 지원하는 확장 마크다운 문법. 표준 마크다운 문법은 마크다운 가이드를 참고하세요.Extended markdown syntax exclusive to Leaf. For standard markdown, see the Markdown Guide.

코넬 노트Cornell Notes Leaf Only

코넬 대학교에서 개발한 3단 노트 포맷. 핵심어(Cue), 본문 노트(Note), 요약(Summary)을 구분하여 학습 효과를 높인다. Leaf에서 마크다운으로 직접 작성하고 렌더링할 수 있다.A 3-section note format developed at Cornell University. Separates keywords (Cue), main notes (Note), and summary to improve learning. Write and render directly in Leaf's markdown.

cornell-note.mdMarkdown
--- type: cornell title: React Hooks date: 2026-04-19 subject: Frontend --- ::: cue useState useEffect 의존성 배열 ::: ::: note useState는 컴포넌트 상태를 관리한다. useEffect는 사이드 이펙트를 처리한다. 의존성 배열이 비면 마운트 시 1회만 실행. ::: ::: summary Hooks는 함수형 컴포넌트에서 상태와 사이드 이펙트를 관리하는 핵심 API다. :::

칸반 보드Kanban Board Leaf Only

마크다운 파일 안에서 칸반 보드를 만들어 작업을 관리한다. 컬럼별 색상, 체크리스트, 마크다운 서식을 지원한다.Create Kanban boards within markdown files to manage tasks. Supports column colors, checklists, and full markdown formatting in cards.

project-board.mdMarkdown
--- type: kanban columns: [할 일, 진행 중, 완료] --- --- column: 할 일 color: yellow --- ## API 설계 - [ ] 엔드포인트 정의 - [ ] 스키마 작성 --- column: 진행 중 color: blue --- ## DB 마이그레이션 - [x] 스키마 설계 - [ ] 마이그레이션 스크립트

메모 보드Memo Board Leaf Only

Google Keep 스타일의 스티키 노트 보드. 카테고리와 색상으로 분류하고, 카드 안에서 마크다운 서식을 사용할 수 있다.Google Keep-style sticky note board. Categorize with colors, use full markdown inside cards.

ideas.mdMarkdown
--- type: memo categories: [아이디어, 리서치, 버그] --- --- color: yellow category: 아이디어 --- ## 다크모드 자동 전환 시스템 설정 연동해서 자동으로 전환 --- color: pink category: 버그 --- ## 테이블 정렬 깨짐 3열 이상에서 우측 정렬 안 됨

확장 알림 블록Extended Alert Blocks Extended

GitHub 스타일의 기본 알림(NOTE, TIP, WARNING, IMPORTANT, CAUTION) 외에 Leaf만의 확장 알림 20종을 지원한다.Beyond GitHub's standard alerts (NOTE, TIP, WARNING, IMPORTANT, CAUTION), Leaf supports 20 additional alert types.

기본 + 확장Standard + ExtendedMarkdown
> [!NOTE] > 참고할 내용 > [!TODO] ← Leaf 확장 > 처리할 작업 > [!BUG] ← Leaf 확장 > 알려진 버그 > [!DEFINITION] ← Leaf 확장 > 용어 정의 > [!THEOREM] ← Leaf 확장 > 수학 정리

확장 알림 카테고리Extended Alert Categories

DEFINITION용어의 정의를 설명합니다Explains the definition of a term
TODO처리해야 할 작업입니다A task that needs to be completed
BUG알려진 버그에 대한 설명입니다Description of a known bug

학습용: DEFINITION, EXAMPLE, QUESTION, SUMMARY, KEYPOINT, THEOREM, PROOF
계획용: TODO, GOAL, DEADLINE, PROGRESS

프론트매터 속성Frontmatter Properties Extended

Leaf는 YAML 프론트매터에 확장 속성을 지원한다. 유용성 별점, 상태 관리, 댓글 등을 메타데이터로 관리할 수 있다.Leaf supports extended frontmatter properties. Manage usefulness ratings, status, comments and more as metadata.

frontmatterYAML
--- title: 프로젝트 설계 문서 tags: [architecture, design, v2] date: 2026-04-19 status: active # active|draft|outdated|archived usefulness: 4 # 1~5 별점 aliases: [설계문서, arch-doc] comments: - "2026-04-15: 1차 리뷰 완료" - "팀 공유 필요" ---

ABC 악보ABC Music Notation Leaf Only

ABC 문법으로 악보를 렌더링하고 재생까지 할 수 있다. BPM 조정, 일시정지, 노트 하이라이트를 지원한다.Render and play sheet music with ABC notation. Supports BPM adjustment, pause, and note highlighting.

music.mdMarkdown
```abc X:1 T:Twinkle Twinkle Little Star M:4/4 L:1/4 K:C C C G G|A A G2|F F E E|D D C2|] ```

인터랙티브 테이블Interactive Tables Extended

표준 마크다운 테이블 문법을 쓰되, Leaf에서는 셀 클릭 편집, 우클릭 행/열 추가/삭제, 정렬 제어, 수식까지 지원한다.Standard markdown table syntax, but Leaf adds click-to-edit, right-click row/column operations, alignment control, and formulas.

표준 마크다운 문법(제목, 목록, 링크, 이미지, 코드 블록 등)은 마크다운 가이드를 참고하세요. 이 페이지에서는 Leaf에서만 동작하는 확장 문법만 다룹니다.For standard markdown syntax (headings, lists, links, images, code blocks, etc.), see the Markdown Guide. This page covers only Leaf-exclusive extensions.
